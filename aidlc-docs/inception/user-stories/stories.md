## Group 1: モード選択・スタートフロー

---

### US-001: モード選択

**As a** 作文の宿題がある児童（さくら）
**I want to** 「ヒントを使うモード」と「直接書くモード」を選べるようにしたい
**So that** 自分の状況（ヒントが必要か、すぐ書けるか）に合った方法で始められる

**Phase 0 優先: ★**

#### Acceptance Criteria

```
シナリオ 1: プランモードを選択する
  Given: ホーム画面（モード選択画面）を開いている
  When: 「プランモード（ヒントを使って書く）」を選択する
  Then: 作文種別選択画面（F-01）に遷移する

シナリオ 2: フリーモードを選択する
  Given: ホーム画面（モード選択画面）を開いている
  When: 「フリーモード（直接書く）」を選択する
  Then: 枚数設定を経て原稿用紙画面（F-05）に直接遷移する
```

---

### US-002: 作文種別選択

**As a** 読書感想文や行事の感想を書く必要がある児童（さくら）
**I want to** 書く作文の種類を選びたい
**So that** 自分の課題に合ったヒント質問を受けられる

**Phase 0 優先: ★**

#### Acceptance Criteria

```
シナリオ 1: 種別を選択するとヒントが切り替わる
  Given: プランモードのスタートフロー画面を開いている
  When: 「読書感想文」「行事の感想」「自由作文」のいずれかを選択する
  Then: 選択した種別に対応するヒント質問（F-03）に切り替わる

シナリオ 2: 選択前は次のステップに進めない
  Given: 作文種別選択画面を開いている
  When: 種別を選択していない状態で次へ進もうとする
  Then: 次のステップへの遷移がブロックされる（または選択を促すメッセージが表示される）
```

---

### US-003: 学年設定

**As a** 特定の学年の児童（さくら）
**I want to** 自分の学年を設定したい
**So that** 学年に合った語彙・難易度のヒント質問を受けられる

**Phase 0 優先: ★**

#### Acceptance Criteria

```
シナリオ 1: 学年を設定するとヒント内容が変わる
  Given: プランモードのスタートフロー画面を開いている
  When: 学年（小1〜高3）を選択する
  Then: 選択した学年に応じたヒント質問の内容・語彙に切り替わる

シナリオ 2: 学年は次回起動時も保持される
  Given: 前回の利用時に学年を設定している
  When: アプリを次回起動する
  Then: 前回設定した学年がデフォルト値として表示される（localStorageから復元）
```

---

### US-004: 質問形式ヒント

**As a** 何を書けばいいかわからない児童（さくら）
**I want to** 「どこで？」「誰と？」「どんな気持ち？」などの質問に答えたい
**So that** 書くべき内容が整理されて、作文を書き始められる

**Phase 0 優先: ★**

#### Acceptance Criteria

```
シナリオ 1: 質問が3問表示される
  Given: 作文種別と学年を設定してヒント画面に進んでいる
  When: ヒント画面が表示される
  Then: 選択した種別に対応する固定3問のヒント質問が表示される

シナリオ 2: 全問回答後に原稿用紙へ進める
  Given: ヒント画面で3問すべてに回答している
  When: 「原稿用紙を作成」ボタンを押す
  Then: 入力した回答内容を参照できる状態で原稿用紙画面に遷移する

シナリオ 3: 未回答でも原稿用紙へ進める（任意回答）
  Given: ヒント画面で一部の質問を未回答のまま
  When: 「原稿用紙を作成」ボタンを押す
  Then: 警告なしで原稿用紙画面に遷移できる（ヒントは任意入力）
```

---

### US-005: イルカキャラクター表示

**As a** 作文が苦手で不安な児童（さくら）
**I want to** イルカキャラクターが質問を案内してくれるのを見たい
**So that** 楽しい雰囲気でヒント入力を続けられる

**Phase 0 優先: ★**

#### Acceptance Criteria

```
シナリオ 1: ヒント画面でイルカが表示される
  Given: プランモードのヒント入力画面を開いている
  When: 画面が表示される
  Then: イルカキャラクターが画面内に表示され、質問を提示している

シナリオ 2: 文字数が目標に近づくとイルカがリアクションする
  Given: 原稿用紙画面で作文を書いている
  When: 入力文字数が目標文字数の90%以上に達する
  Then: イルカがさりげないリアクション（アニメーションまたはコメント）を示す

シナリオ 3: 通常の入力中はイルカが静止している
  Given: 原稿用紙画面で作文を入力している（目標90%未満）
  When: 文字を入力・削除する
  Then: イルカはアニメーションせず静止している（速度優先）
```

---

### US-006: フリーモード開始

**As a** すでに書く内容が決まっている児童（さくら）
**I want to** ヒント入力をスキップして原稿用紙に直接進みたい
**So that** 時間をムダにせず書き始められる

**Phase: Phase 1（5/4 MVP ローンチ）**

#### Acceptance Criteria

```
シナリオ 1: フリーモード選択で枚数設定を経て原稿用紙へ直接遷移する
  Given: モード選択画面でフリーモードを選択している
  When: 枚数（2/4/8/10枚）を選択して開始ボタンを押す
  Then: ヒント入力画面を経由せず原稿用紙画面に遷移する

シナリオ 2: フリーモードから途中でプランモードに戻れる
  Given: フリーモードで原稿用紙画面を開いている
  When: 「ヒントを使って書く」ボタンを押す（またはホームに戻る）
  Then: プランモードのスタートフロー画面に遷移できる
```

## Group 2: 原稿用紙エンジン

---

### US-007: 縦書き原稿用紙UI

**As a** 日本語の作文課題がある児童（さくら）
**I want to** 学校の原稿用紙と同じ縦書きグリッドで作文を書きたい
**So that** 学校提出に適した形式で書けて、印刷結果も本物の原稿用紙に見える

**Phase 0 優先: ★（コア機能）**

#### Acceptance Criteria

```
シナリオ 1: 入力文字がリアルタイムで縦書きグリッドに反映される
  Given: 原稿用紙画面（タブレット横向き）を開いている
  When: 右側の横書き入力エリアにテキストを入力する
  Then: 入力文字が左側の縦書きグリッド（20列×20行、1文字1マス）にリアルタイムで反映される

シナリオ 2: グリッドが正しい仕様で表示される
  Given: 原稿用紙画面を開いている
  When: 画面が表示される
  Then: 茶色系罫線・白地・ゴシック体の20×20グリッドが表示される（学校配布の原稿用紙に近い色調）

シナリオ 3: 400字（1ページ）を超えると次ページに移行できる
  Given: 1ページに400字入力済みの状態
  When: さらに文字を入力する
  Then: 次ページに文字が続けて表示され、ページナビゲーションボタンで移動できる

シナリオ 4: タブレット横向きレイアウトが崩れない
  Given: タブレット（iPad等）を横向きで使用している
  When: 原稿用紙画面を表示する
  Then: 左側グリッド・右側入力エリアのスプリットレイアウトが崩れず表示される
```

---

### US-008: タイトル・名前欄

**As a** 作文を学校に提出する児童（さくら）
**I want to** タイトルと自分の名前を入力したい
**So that** 提出物として完成した形になる

**Phase 0 優先: ★**

#### Acceptance Criteria

```
シナリオ 1: タイトルと名前を入力できる
  Given: 原稿用紙画面を開いている
  When: タイトル欄と氏名欄にテキストを入力する
  Then: 入力した内容が原稿用紙の所定の位置（上部）に表示される

シナリオ 2: タイトル・名前が印刷時にも正しく出力される
  Given: タイトルと氏名を入力している
  When: 印刷プレビューを表示する
  Then: タイトルと氏名が原稿用紙の適切な位置に印刷される
```

### US-009: 枚数設定

**As a** 特定の文字数で作文を書く必要がある児童（さくら）または保護者（まお）
**I want to** 作文の枚数（文字数）を設定したい
**So that** 課題で指定された文字数に対応したページ数で書ける

**Phase 0 優先: ★**

#### Acceptance Criteria

```
シナリオ 1: 開始時に枚数を選択できる
  Given: 原稿用紙を作成しようとしている（プランモードまたはフリーモード）
  When: 枚数（2枚=800字 / 4枚=1600字 / 8枚=3200字 / 10枚=4000字）を選択する
  Then: 選択した枚数に対応するページ数の原稿用紙が用意される

シナリオ 2: 選択した枚数が文字数カウントの目標値になる
  Given: 枚数を選択して原稿用紙画面を開いている
  When: 文字数カウンターが表示される
  Then: 選択した枚数×400字が目標文字数として表示される
```

---

### US-010: 文字数カウント

**As a** 指定文字数を満たす必要がある児童（さくら）
**I want to** 今何文字書いたかをリアルタイムで確認したい
**So that** 字数が足りているか・書き過ぎていないかを常に把握できる

**Phase 0 優先: ★**

#### Acceptance Criteria

```
シナリオ 1: 文字入力ごとにカウントが更新される
  Given: 原稿用紙画面で文字を入力している
  When: 文字を1文字入力または削除する
  Then: 「現在の文字数 / 目標文字数」（例: 120 / 400）がリアルタイムで更新される

シナリオ 2: 目標文字数を超えた場合に視覚的に通知する
  Given: 入力文字数が設定した枚数×400字を超えた
  When: 文字数表示を見る
  Then: オーバーした文字数が視覚的に区別されて表示される（色変え等）
```

---

### US-011: 自動保存

**As a** 作文の途中で画面が閉じてしまう可能性がある児童（さくら）
**I want to** 書いた内容が自動的に保存されてほしい
**So that** 途中で止めても次回から続きを書ける

**Phase 0 優先: ★**

#### Acceptance Criteria

```
シナリオ 1: 入力内容がlocalStorageに自動保存される
  Given: 原稿用紙画面で文字を入力している
  When: 文字を入力する（一定間隔または入力変化のたびに）
  Then: ブラウザのlocalStorageに入力内容が保存される

シナリオ 2: 次回アクセス時に保存データが復元される
  Given: 前回の利用で文章を保存している
  When: 同じブラウザ・デバイスで再度アクセスする
  Then: 前回の作文データ（タイトル・本文・学年・種別・枚数設定）が復元される

シナリオ 3: localStorageが使えない場合でも機能が壊れない
  Given: プライベートブラウジングモードなどlocalStorageが制限された環境
  When: アプリを使用する
  Then: エラーが発生せず、手動保存の案内メッセージが表示される（graceful degradation）
```

## Group 3: 出力・サポート

---

### US-012: 印刷機能

**As a** 子どもの作文を学校に提出したい保護者（まお）
**I want to** 完成した作文を実際の原稿用紙と同じ精度で印刷したい
**So that** 学校に提出できる品質の印刷物が得られる

**Phase: Phase 1（5/4 MVP ローンチ）**

#### Acceptance Criteria

```
シナリオ 1: 印刷ボタンでブラウザ印刷ダイアログが起動する
  Given: 原稿用紙画面で作文が入力されている
  When: 「印刷」ボタンを押す
  Then: ブラウザの印刷ダイアログが表示される

シナリオ 2: 印刷結果がA4縦に正しく収まる
  Given: 印刷プレビューを確認している
  When: A4縦サイズで印刷設定を確認する
  Then: 原稿用紙グリッドがA4縦の用紙に正しく収まり、マス目のズレがない

シナリオ 3: 複数ページが正しく印刷される
  Given: 2ページ以上の作文を作成している
  When: 印刷を実行する
  Then: 全ページが順番に印刷される（各ページが1枚のA4用紙に対応）

シナリオ 4: 印刷時にUI要素（ボタン等）が非表示になる
  Given: 印刷プレビューを表示している
  When: 印刷プレビューを確認する
  Then: ナビゲーションボタン・入力エリア等のUI要素が非表示になり、原稿用紙のみが印刷される
```

---

### US-013: 原稿用紙の使い方ガイド

**As a** 原稿用紙のルールをよく知らない児童（さくら）
**I want to** 句読点の位置・段落の書き方などのヒントを見たい
**So that** 正しい形式で作文を書ける

**Phase: Phase 1（5/4 MVP ローンチ）**

#### Acceptance Criteria

```
シナリオ 1: 使い方ガイドが原稿用紙画面から確認できる
  Given: 原稿用紙画面を開いている
  When: 「使い方」ボタンまたはヘルプアイコンを押す
  Then: 原稿用紙のルール（句読点の位置・段落の字下げ・数字の書き方等）が表示される

シナリオ 2: ガイドが執筆の邪魔をしない表示になっている
  Given: 原稿用紙画面でガイドを表示している
  When: ガイドを確認した後に閉じる
  Then: 元の原稿用紙画面に戻り、入力内容は保持されている
```
