# 要件定義書

## Intent Analysis Summary

| 項目 | 内容 |
|---|---|
| プロジェクト種別 | Greenfield（新規開発） |
| リクエスト種別 | New Project |

## プロダクトミッション

> 作文苦手でも、作文がスルスル書ける体験をつくる

**核心的差別化**：「作文前UX」— 競合は「書くツール」だが、本プロダクトは「書き始めを助けるツール」

### 📐 原稿用紙UI 詳細仕様（F-05）

#### グリッド仕様（MVP）

| 項目 | 仕様 |
|---|---|
| グリッド | 20列 × 20行 = **400字詰め**（1文字1マス固定） |
| 書き方向 | **縦書き**（top→bottom、right→left） |
| 印刷向き | **A4縦** |
| 罫線色 | **茶色系**・白地（学校配布の原稿用紙に近い色調） |
| フォント | **ゴシック体**（視認性優先・子ども向け） |
| 印刷精度 | 実際の原稿用紙と見分けがつかないレベル（余白バランス厳守） |

#### 画面レイアウト（タブレット横向き基本）

```
┌─────────────────────────────────────────┐
│  [←前ページ]  1/2ページ  [次ページ→]  │
├──────────────────────┬──────────────────┤
│                      │                  │
│   縦書き原稿用紙     │  横書き入力エリア│
│   グリッド（参照用） │  （テキスト入力）│
│   ┌─┬─┬─┬─┬─┐    │                  │
│   │ │ │ │ │ │    │  ここに文字を    │
│   ├─┼─┼─┼─┼─┤    │  入力すると      │
│   │ │ │ │ │ │    │  左のグリッドに  │
│   └─┴─┴─┴─┴─┘    │  リアルタイム反映│
│                      │                  │
│   文字数: 120/400    │   [印刷] [保存]  │
└──────────────────────┴──────────────────┘
```

#### 入力・編集仕様

| 項目 | 仕様 |
|---|---|
| 入力方式 | 横書きテキストエリアに入力 → 縦書きグリッドにリアルタイム反映 |
| 縦書き直接入力 | MVPでは採用しない |
| 文字修正 | 通常のバックスペースで削除可（特別な演出なし） |
| ページ移動 | 1ページずつ表示・ボタンでページ移動（スクロールなし） |
| 自動保存 | ブラウザ localStorage に自動保存 |

#### キャラクター演出

| タイミング | 演出 |
|---|---|
| 文字入力中 | アニメーションなし（速度優先） |
| 完成に近づいたとき | イルカがさりげなくリアクション |

| 12×12 | 144字 | A4横 | 低学年（小2〜3） |

## 非機能要件

### パフォーマンス

| 項目 | 要件 |
|---|---|
| 初期表示速度 | LCP（Largest Contentful Paint）2.5秒以内 |
| インタラクション | タブレット・スマホで快適に動作 |
| 印刷品質 | A4横/縦で正しく出力される |

### SEO / AIO（重要：成功指標に直結）

| 項目 | 要件 |
|---|---|
| ターゲットキーワード | 「原稿用紙 オンライン」「作文 原稿用紙 アプリ」「読書感想文 書き方」等 |
| レンダリング | SSR必須（Googleのクローラーがコンテンツを読める状態） |
| メタ情報 | title / description / OGP を各ページに設定 |
| AIO対応 | 構造化データ（Schema.org）でAIエンジンへの情報提供を最適化 |
| サイトマップ | sitemap.xml を自動生成・送信 |

### プライバシー（推奨設計）

**推奨アプローチ：MVP段階は「データを持たない設計」でプライバシーリスクをゼロに近づける。**

理由：子ども向けサービスはプライバシー規制（個人情報保護法・COPPA等）が厳格。MVPで個人データを持たないことで、規制対応コストをゼロにしながら早期ローンチが可能。

| 項目 | MVP設計 | Phase 2設計 |
|---|---|---|
| サーバー側データ保存 | ❌ なし | ✅ アカウント導入時に追加 |
| ブラウザ保存 | localStorage（下書き自動保存のみ） | ユーザーID紐付け |
| 個人情報収集 | ❌ なし | 保護者メールのみ |
| プライバシーポリシー | ✅ 簡易版を必ず用意（法的義務） | 本格版に更新 |
| Cookieバナー | ❌ 不要（トラッキングCookieなし） | Google Analytics導入時に追加 |

### セキュリティ（推奨設計）

**推奨：セキュリティ拡張を適用するが、MVPに非該当なルールはN/Aとして処理する。**

子ども向けの公開Webサービスは、アカウントがなくても基本的なセキュリティ対策が必要（悪意あるスクリプトの埋め込み等を防ぐ）。

| セキュリティルール | MVP適用 | 理由 |
|---|---|---|
| SECURITY-04（HTTPセキュリティヘッダー） | ✅ 適用 | Next.js middlewareで容易。XSS等を防ぐ |
| SECURITY-05（入力バリデーション） | ✅ 適用 | テキスト入力はすべてバリデーション |
| SECURITY-09（ハードニング） | ✅ 適用 | デフォルト設定の無効化・エラー情報非表示 |
| SECURITY-01（暗号化） | N/A | MVPはサーバーDBなし |
| SECURITY-06（最小権限） | N/A | MVPはバックエンドなし |
| SECURITY-12（認証管理） | N/A | MVPはアカウントなし |
| その他 | N/A | 同様の理由 |

## 技術スタック推奨

**推奨：Next.js（TypeScript）+ Vercel**

### 推奨理由

| 技術 | 選定理由 |
|---|---|
| Next.js（App Router） | SSRによるSEO最適化が必須。APIルートで将来のバックエンドも同一フレームワークで対応可能 |
| TypeScript | 型安全で一人開発でもバグを早期検出。AI（Claude Code等）との相性が良い |
| Vercel | Next.jsのデプロイに最適。無料枠で十分（月100GB帯域）。GitHub連携でCI/CD自動化 |
| localStorage | MVP段階のデータ永続化。サーバー不要で即日デプロイ可能 |

### 将来拡張パス

```
Phase 1 MVP: Next.js + Vercel + localStorage
Phase 2:     + Supabase（Auth + DB）or PlanetScale
Phase 3:     + AWS（高トラフィック対応・負荷分散が必要になった場合）
```
