## Q: 入力エリアと原稿用紙グリッドの配置関係

「横書きで入力 → 縦書きグリッドにリアルタイム反映」という方式を採用しますが、
**タブレット画面上でどう並べるか**が未決定です。

タブレットでソフトキーボードを出すと画面の約半分が隠れます。
その状態でも「グリッドを見ながら入力できる」レイアウトが必要です。

A) 上：縦書きグリッド（参照用）／ 下：横書き入力エリア
　　→ キーボードが出ると上のグリッドが圧縮されるが常に見える

B) 左：縦書きグリッド（参照用）／ 右：横書き入力エリア（横並び）
　　→ タブレット横向き時に最適。縦向きでは幅が狭くなる

C) 入力エリアのみ表示（グリッドは印刷プレビューボタンで確認）
　　→ 入力に集中できる。グリッド表示は別画面扱い

D) まだ決めていない / 提案してほしい

[Answer]:B:左に縦書き原稿用紙グリッド、右に横書き入力エリアを配置する。学校配布のキーボード付きタブレットでの利用を想定し、横向きレイアウトを基本設計とする。常に原稿用紙を見ながら入力できる構成とする。

## Q2: 原稿用紙のグリッドサイズ・書き方向の切り替え

以下の規格をすべてサポートしたい旨を確認しました。

**確認済み原稿用紙の規格（すべて縦書き）：**

| 規格 | 字数 | 印刷向き | 対象 |
|---|---|---|---|
| 9×9 | 81字 | A4横 | 低学年（小1〜2） |
| 12×12 | 144字 | A4横 | 低学年（小2〜3） |
| 15×15 | 225字 | A4横 | 低〜中学年 |
| 20×20 | 400字 | A4縦 | 小3〜高校（標準） |

✅ **全規格が縦書きのため、レンダリングエンジンは1種類で統一可能。**
グリッドサイズと印刷時のページ向き（A4横/縦）だけが変わります。

A) 学年設定と自動連動（学年に応じてグリッドと書き方向を自動切り替え）
　　→ 最もUXが自然。ただし横書き・縦書き両方の実装が必要

B) 作文用紙の種類をユーザーが手動で選択（学年と切り離す）
　　→ 課題の指定に柔軟対応できる。選択肢に規格を明示

C) 両方（学年でデフォルト設定、手動でオーバーライドも可）
　　→ 最も柔軟。実装コストは最大

D) MVP は 20×20（400字・縦書き）のみ。低学年規格は Phase 2 対応
　　→ 開発コスト最小。4月デモに最も安全

X) Other (please describe after [Answer]: tag below)

[Answer]:MVPは 20×20（400字・縦書き・A4縦）のみ実装する。
低学年規格（200字・横書き等）は Phase 2 で対応する。
レンダリングエンジンは1種類に統一し、完成優先とする。
